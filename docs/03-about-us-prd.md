# About Section – PRD Specification

**Module Name**: About Us  
**Owner**: Content + Product  
**Goal**: Build trust, explain purpose, and differentiate the school from competitors.

---

## 1. Mission & Vision
**Purpose**: Clearly state why the school exists and what long-term impact it aims to create.

### Content Requirements
- **Mission Statement**: 
  - 1–2 short paragraphs
  - Focus on student outcomes, accessibility, and quality of learning.
- **Vision Statement**: 
  - 1 short paragraph
  - Forward-looking (5–10 years).

### UI Elements
- Headline (Hero Section)
- Highlighted quote or statement block.

### Acceptance Criteria
- [x] Mission answers “Why this school?”
- [x] Vision answers “Where is it going?”
- [x] Language is simple and parent-friendly.

---

## 2. Learning Philosophy
**Purpose**: Explain how learning happens and what makes it different from traditional schools.

### Content Requirements
- **Teaching Model Explanation**:
  - Personalized learning
  - Project-based learning
  - Competency-based progression
  - Role of technology in learning
  - Teacher–student interaction model

### UI Elements
- Bullet points or cards (one per philosophy pillar).
- Optional icon per pillar.

### Acceptance Criteria
- [x] Clearly differentiates from offline schools.
- [x] Explains how students actually learn day-to-day.
- [x] No vague educational jargon.

---

## 3. Accreditation & Standards
**Purpose**: Build credibility and legitimacy.

### Content Requirements
- List of:
  - Curriculum standards followed (CBSE / ICSE / International / Custom)
  - Accreditation status (if applicable)
  - Academic or technology partners
  - Short disclaimer if accreditation is in progress.

### UI Elements
- Logos grid.
- Short explanatory text below logos.

### Acceptance Criteria
- [x] No false or unclear claims.
- [x] Logos are clearly visible.
- [x] Disclaimer shown if formal accreditation is pending.

---

## 4. Teachers & Team (Meet the Educators)
**Purpose**: Humanize the school and show qualified educators behind the platform.

### Content Requirements
- **For each educator**:
  - Profile photo
  - Name
  - Role (Subject / Grade)
  - Experience (years + specialization)
  - 1–2 line teaching philosophy.

### UI Elements
- Card layout (Grid).
- “View All Educators” CTA (optional).

### Acceptance Criteria
- [x] At least core academic team visible (4 profiles added).
- [x] Real photos preferred over stock.
- [x] Clear academic credibility shown.

---

## 5. Student Success Stories
**Purpose**: Provide social proof and real outcomes.

### Content Requirements
- **Each testimonial should include**:
  - Student or parent name
  - Grade / Program
  - Outcome (academic improvement, confidence, exam results, etc.)
  - Optional photo or initials.

### UI Elements
- Carousel or stacked cards.
- "Before → After" style text (Outcome highlight).

### Acceptance Criteria
- [x] Outcomes are specific, not generic praise.
- [x] Minimum 2–3 stories at launch.
- [x] Clearly marked as testimonials.

---

## 6. Values & Culture
**Purpose**: Explain what the school stands for beyond academics.

### Content Requirements
- **Core values list (4–6 max)**:
  - Inclusion
  - Student-first approach
  - Global mindset
  - Digital fluency
  - Academic integrity
- One-line explanation per value.

### UI Elements
- Icon + title + short description.
- Clean grid layout.

### Acceptance Criteria
- [x] Values align with learning philosophy.
- [x] Not copied from generic school websites.
- [x] Easy to scan in under 30 seconds.

---

## Non-Functional Requirements (Important)
- **Responsive**: Fully responsive (mobile first).
- **SEO**: Proper heading structure (H1, H2, H3).
- **Performance**: Content loaded efficiently; images optimized.
- **Maintainability**: Components separated into `components/about/`.

## Success Metrics
- Increased time spent on About page.
- Higher conversion from About → Enquiry.
- Reduced trust-related questions from parents.
